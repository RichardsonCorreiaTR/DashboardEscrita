1. Pré-requisitos (máquina TR)

ItemObservação
Node.js 18+Baixar em nodejs.org — marcar Add to PATH na instalação
GitBaixar em git-scm.com (opções padrão)
Driver ODBC Sybase ASA 9Já instalado nas máquinas com Domínio/SGD
DSN ODBC pbcvs9Painel de Controle → ODBC (64 bits) → DSN de Sistema
Importante: sem o DSN pbcvs9, o dashboard abre, mas não consulta dados do banco SGD.

2. Baixar o projeto

1 Abrir PowerShell (usuário normal, sem admin).

2 Executar os comandos abaixo:

cd "C:\1 - A\B\Programas"
git clone https://github.com/RichardsonCorreiaTR/DashboardEscrita.git
cd DashboardEscrita

Se o repositório for privado, é necessário convite de acesso no GitHub (falar com Richardson).

Pasta alternativa: pode clonar em qualquer diretório; ajuste o caminho nos comandos.

3. Instalar dependências

$env:PATH = "C:\Program Files\nodejs;$env:PATH"
npm install

Aguardar 1–2 minutos até concluir.

4. Iniciar o servidor local

npm start

Quando aparecer Dashboard rodando em http://localhost:4000, o servidor está ativo.

Para encerrar: pressionar Ctrl + C no terminal.

O terminal precisa permanecer aberto enquanto usar o dashboard.

5. Acessar no navegador

8. Problemas comuns

ProblemaSolução
npm não reconhecidoReinstalar Node.js com "Add to PATH" ou usar o comando do passo 3
Erro de conexão ODBCVerificar DSN pbcvs9 no ODBC 64 bits
Página não abreConfirmar que npm start está rodando no terminal
Dados desatualizadosClicar em Atualizar ou reiniciar o servidor

9. Verificar DSN ODBC (se necessário)

  1. Painel de Controle → Ferramentas Administrativas → Fontes de Dados ODBC (64 bits)
  2. Aba DSN de Sistema → confirmar entrada pbcvs9
  3. Se não existir, solicitar configuração ao suporte de TI / Richardson

10. Inicialização automática (opcional)

  1. Pressionar Win + R, digitar shell:startup e Enter
  2. Criar atalho para npm start na pasta do projeto (ou editar start-server.bat com o caminho correto)
